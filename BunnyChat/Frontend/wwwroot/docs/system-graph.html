BunnyChat System Graph

Tai lieu nay mo ta chi tiet toan bo he thong hien tai: frontend ASP.NET MVC, API backend, MongoDB, SignalR realtime chat, SignalR realtime ket ban, chuong thong bao va cac ham JS chinh. Muc tieu la nhin vao day biet file nao lam gi, ham nao goi API nao, event nao chay qua SignalR, response nao duoc UI render.

Tong quan Luong nghiep vu Function graph SignalR Frontend Backend API Response File map Ghi chu

1. Tong Quan Kien Truc

View /Chat

Razor view render layout chat, chuong thong bao, search user, danh sach nhom va ban be.

JS ChatAction.js

Dieu phoi UI, goi API, ket noi SignalR, render tin nhan va loi moi ket ban.

Auth Api_Fetch.js

Gan Bearer token, tu refresh access token khi server tra 403.

API Controllers

User, Friend, Chat controller xu ly request tu frontend.

Realtime ChatHub

SignalR group theo user va conversation de gui event tuc thoi.

Data MongoDB

Luu users, sessions, friends, conversations, messages.

3. Function Graph: Ham Goi Ham Nao

Khoi tao trang

loadChatData() -> fetchData() -> apiFetch() -> render groups/friends/badge -> connectRealtime().

Search user

searchUsers() -> API.search() -> createConversationCard(type="search") -> click card mo profile, click nut gui loi moi.

Gui loi moi ket ban

sendFriendRequestFromCard() -> FriendController.SendFriendRequest() -> Mongo friends -> SignalR friend-request-created.

Mo chuong thong bao

Click bell -> openFriendModal() -> setFriendRequestTab() -> renderFriendRequests() -> createFriendRequestCard().

Accept friend

acceptFriendRequest() -> API accept -> update state.friendRequests va state.friends -> renderFriends().

Decline friend

declineFriendRequest() -> API decline -> remove request khoi state -> cap nhat modal va badge.

Chon conversation

selectConversation() -> API.messages() -> renderMessages() -> joinConversation() -> markActiveConversationAsSeen().

Gui message

Submit form -> API.sendMessage() -> ChatController.SendMessage() -> SignalR new-message -> appendRealtimeMessage().

4. SignalR Realtime

SignalR endpoint

Program.cs map: app.MapHub<ChatHub>("/chatHub").

JWT lay tu query string access_token khi frontend mo websocket.

User group

ChatHub.UserGroup(userId) tra group dang user:{id}.

Dung cho event ca nhan: loi moi ket ban moi, accept, decline.

Conversation group

ChatHub.ConversationGroup(conversationId) tra group dang conversation:{id}.

Dung cho tin nhan realtime va trang thai da xem trong mot cuoc tro chuyen.

Event Server phat tu file/ham Client nghe o dau Payload chinh UI thay doi
new-conversation ChatController.CreateConversation() connectRealtime() conversation da format, co userName, members, directUserId Them hoac cap nhat group/direct chat trong sidebar.
new-message ChatController.SendMessage() connectRealtime() { message, conversation, unreadCounts } Cap nhat last message va append bubble neu dang mo dung conversation.
read-message ChatController.MarkAsSeen() connectRealtime() { conversation, lastMessage } Cap nhat unread/seen cho conversation.
friend-request-created FriendController.SendFriendRequest() connectRealtime() request gom id, message, createdAt, user Chuong thong bao tang badge, modal loi moi them card moi.
friend-request-accepted FriendController.AcceptFriendRequest() connectRealtime() { requestId, user } Nguoi gui duoc cap nhat danh sach ban be realtime.
friend-request-declined FriendController.DeclineFriendRequest() connectRealtime() { requestId } Xoa request khoi tab da gui.

8. Response Quan Trong

User mini object

Dung trong friends, requests, members, search/profile.

{ id, username, userName, displayname, avatarUrl }

userName duoc them de frontend dung thong nhat, van giu username de khong pha code cu.

Conversation object

{ id, type, name, directUserId, username, userName, memberCount, members, lastMessage, lastMessageAt, unreadCount }

Direct chat dung directUserId de ghep conversation voi card ban be.

Friend request object

{ id, message, createdAt, status, user }

user la nguoi con lai trong request: tab da nhan la nguoi gui, tab da gui la nguoi nhan.

Message object

{ id, conversationId, senderId, content, imgUrl, isMine, createdAt, updatedAt }

isMine giup render bubble ben phai/trai.

10. Ghi Chu De Nho

  • API la request HTTP tu frontend toi controller.
  • Event la du lieu server day realtime qua SignalR.
  • state nam trong ChatAction.js, dung de render UI.
  • Badge chuong chi dem loi moi da nhan, khong dem loi moi da gui.
  • userName duoc tra them, nhung username van giu de tuong thich.
  • Data server UI hien render tu API, khong con dung danh sach message/user mau co dinh.
